架构变革：砍掉了什么

旧方案（重型工程）

  • PostgreSQL 建表 + Alembic 数据库迁移
  • FastAPI 后端开发 2000+ 行代码
  • MCP Server 开发
  • 前端管理后台
  • Redis 消息队列
  • Celery 异步任务
  • 需要 5-6 人 IT 团队
  • 周期 12 周

新方案（轻量拼接）

  • 飞书多维表格 = 数据库 + 管理后台 + 图片库
  • Dify = AI 引擎 + Prompt 管理
  • n8n = 工作流调度 + API 胶水
  • Trae = 写脚本（你说需求，它写代码）
  • Docker 只跑 n8n + Python 脚本
  • 你一个人操作，不用 IT 背景
  • 周期 2-3 周

安装 Docker Desktop + 启动 n8n

Docker n8n
目标： 在你电脑上装好 Docker，把 n8n 跑起来，浏览器能打开 n8n 控制台。

操作步骤

  1. 下载 Docker Desktop：https://www.docker.com/products/docker-desktop/
  2. 安装完成后重启电脑（Docker 要求 WSL2 或 Hyper-V）
  3. 打开终端（PowerShell 或 CMD），进入项目目录：
    cd AI-Operations-Project\docker
  4. 复制环境变量文件：copy ..\.env.example ..\.env
  5. 启动 n8n：docker compose up -d n8n
  6. 浏览器打开 http://localhost:5678 → 用 .env 里设的账号密码登录
✅ 验收标准： n8n 网页能正常打开，创建第一个测试工作流（随便拖一个 HTTP Request 节点），点击 Execute 能跑通。

建立飞书多维表格 — 你的产品数据库

飞书
目标： 在飞书里建一张表，等同于传统项目的数据库 + 管理后台。字段覆盖产品从入库到上架的全生命周期。

操作步骤

  1. 打开飞书 → 新建多维表格 → 命名"EMKE产品主表"
  2. 参考 feishu/table_schema.json 里的字段定义，逐列创建：
    基础信息：SKU（文本）、品类（单选：LED浴室镜/暖气片/淋浴房/…）、中文品名（文本）
    AI 生成：金蝶原始数据（多行文本）、德语通用文案（多行）、英语通用文案（多行）、法语通用文案（多行）、Amazon DE Listing（多行）、Amazon FR/UK Listing、Shopify Listing、Otto Listing
    图片：主图（图片）、详情图（图片·多张）、场景图（图片·多张）
    状态管理：审核状态（单选：草稿→待审核→已通过→已上架）、上架平台（多选）、上架状态（多行·JSON）
    广告：广告状态（单选）、最近广告建议（多行）
    其他：备注（多行）
  3. 在飞书开放平台（open.feishu.cn）创建自建应用 → 获取 App ID 和 App Secret → 填入 .env
  4. 在飞书表格 URL 中找到 base/ 后面的那串 ID → 填入 FEISHU_BITABLE_ID
✅ 验收标准： 表格建好，至少包含 22 个字段。飞书应用已授权，拿到 API 凭证。

Dify 编排 AI 文案工作流

Dify
目标： 在 Dify 建一个 AI 工作流，输入金蝶产品参数，输出地道德语通用文案 + 各平台适配版 + 合规检查。

操作步骤

  1. 注册 Dify Cloud（免费）：https://cloud.dify.ai
  2. 创建应用 → 选择"工作流"类型 → 命名"EMKE文案引擎"

Dify 工作流节点配置（3 个核心节点）

节点 1 — 德语通用文案生成（LLM 节点）
System Prompt：
"你是 EMKE 品牌的资深德语电商文案。EMKE 是德国本土卫浴品牌，定位中高端，主营 LED 浴室镜、暖气片、淋浴房、陶瓷盆、水龙头、淋浴柱、马桶、浴室柜。请根据输入的产品参数，撰写地道德语产品文案。风格要求：专业但不生硬、突出德国制造工艺、包含关键卖点、适合欧洲消费者阅读习惯、字数 200-300 词："

节点 2 — 平台适配 Listing（LLM 节点）
System Prompt：
"你是一位跨境电商 Listing 优化专家。请将输入的德语产品文案，分别改编为以下平台格式：1) Amazon DE — Title（200字符内，含核心关键词）+ 5 条 Bullet Points（每条约 200 字符）+ 产品描述。遵循 Amazon A9 算法规则，关键词自然分布。2) Shopify — 吸引人的标题 + 有说服力的产品描述 + SEO 友好的元描述。3) Otto — 遵循 Otto 平台简洁风格。4) Amazon FR/UK — 分别翻译为法语和英语。"

节点 3 — 违禁词筛查（LLM 节点）
System Prompt：
"请检查以下产品文案，识别并标注：1) 违反欧盟产品合规声明的表述（如未经证实的医疗效果宣称）2) 与德国消费者保护法冲突的夸张宣传 3) 平台禁用词（如 Amazon 禁止的'best'、'#1'等绝对化用语）。输出：合规检查通过 / 需要修改，如需修改请明确指出问题词语和建议替换。"
  1. 测试工作流：输入一个真实产品参数（如 LED 浴室镜 EMKE-BS-001，60×80cm，三色温，IP44防水），看输出质量
  2. 发布 → 发布为 API → 复制 API Key → 填入 .env 的 DIFY_API_KEY
✅ 验收标准： Dify 工作流能接收产品参数，输出德/英/法三种语言的通用文案和平台文案，合规检查能标出问题词。

用 Trae 生成金蝶适配器脚本

Trae Docker
目标： 让 Trae 写一个 Python 脚本，定时从金蝶 ERP 拉取新增/变更的产品数据，推送给 n8n。

操作步骤

  1. 打开 Trae，打开项目文件夹 scripts/kingdee_adapter/
  2. 把金蝶 ERP 的 API 文档（或接口说明）准备好
我需要对金蝶 ERP 系统做数据对接。请写一个 Python 脚本（kingdee_adapter.py），功能如下： 1. 通过 HTTP 请求调用金蝶 API，获取最近 24 小时内新增或修改的产品数据 2. 金蝶 API 地址从环境变量 KINGDEE_API_URL 读取，API Key 从 KINGDEE_API_KEY 读取 3. 解析金蝶返回的 JSON，提取以下字段： - SKU 编码 - 产品名称（中文） - 品类 - 规格参数（长宽高、材质、颜色等） - 价格（欧元） - 库存数量 4. 将提取的数据转为 n8n Webhook 能接收的 JSON 格式 5. 通过 HTTP POST 发送到 n8n 的 Webhook URL（从环境变量 N8N_WEBHOOK_URL 读取） 6. 使用 schedule 库，每 2 小时自动执行一次 7. 添加日志输出，方便排查问题 8. 使用 python-dotenv 读取 .env 配置
  1. Trae 生成代码后，在终端运行测试：cd scripts/kingdee_adapter && python kingdee_adapter.py
  2. 确认日志输出正常，能把测试数据发给 n8n
  3. 把需要的 Python 包追加到 scripts/requirements.txt
✅ 验收标准： 脚本运行后，n8n 的 Webhook 节点能收到金蝶产品数据。

n8n 串联全流程

n8n Dify 飞书
目标： 在 n8n 里搭建第一个完整工作流：接收金蝶数据 → 调 Dify 生成文案 → 写入飞书。

n8n 工作流节点链（拖拽连线即可）

节点 1：Webhook（触发器）
→ 路径：/kingdee-product
→ 方法：POST
这个节点接收第 4 步脚本推送过来的金蝶数据。

节点 2：HTTP Request → Dify API
→ 方法：POST
→ URL：https://api.dify.ai/v1/workflows/run
→ Headers：Authorization: Bearer {{ $env.DIFY_API_KEY }}
→ Body：把金蝶的产品参数拼成 Dify 要求的 inputs 格式
这个节点调用第 3 步创建的 Dify 文案工作流。

节点 3：飞书多维表格 → 新增记录
→ 使用飞书节点（n8n 内置飞书集成）
→ 表格：EMKE产品主表
→ 映射：SKU → SKU、品类 → 品类、金蝶原始数据 → 金蝶原始数据、Dify返回的德语文案 → 德语通用文案、英语文案 → 英语通用文案、…
这个节点把 Dify 生成的所有文案回填到飞书表格对应列。
✅ 验收标准： n8n 工作流跑完一遍后，飞书表格里出现一条新记录，所有 AI 生成的文案列都有内容。

图片自动化处理（Trae 脚本）

Trae 飞书
目标： 把产品图片文件夹拖到一个监控目录，脚本自动按规则重命名并上传到飞书表格对应 SKU。
请写一个 Python 图片自动处理脚本（image_processor.py），功能如下： 1. 使用 watchdog 库监控 /app/images 目录 2. 当检测到新文件夹被创建（或放入），文件夹名字就是 SKU（如 EMKE-BS-001） 3. 自动扫描文件夹内的图片（支持 .jpg/.png/.webp） 4. 按文件名排序后，自动重命名为： - 第一张 → MAIN.jpg - 第二、三、四张 → DETAIL_1.jpg, DETAIL_2.jpg, DETAIL_3.jpg - 第五张起 → SCENE_1.jpg, SCENE_2.jpg, ... 5. 使用飞书开放平台 API，将重命名后的图片上传到飞书多维表格： - 表 ID 从环境变量 FEISHU_BITABLE_ID 读取 - 飞书 App ID / Secret 从环境变量读取 - MAIN.jpg 上传到"主图"列 - DETAIL_*.jpg 上传到"详情图"列 - SCENE_*.jpg 上传到"场景图"列 - 根据 SKU 匹配对应行 6. 处理完成后，在日志输出处理结果 7. 使用飞书 SDK（lark-oapi）简化 API 调用
✅ 验收标准： 拖一个以 SKU 命名的图片文件夹进去，飞书表格里对应的行自动出现图片。

Amazon 广告报表自动下载（Trae 脚本）

Trae n8n
目标： 每周一凌晨自动下载 Amazon 广告周报，上传到飞书。
请写一个 Python 脚本（amazon_ads/report_downloader.py），通过 Amazon Advertising API 下载广告报表。要求： 1. 使用 Amazon Ads API（https://advertising-api.amazon.com） 2. 从环境变量读取认证信息 3. n8n 每周一通过 HTTP 触发，传入参数：reportDate（上周一至周日）、campaignType（sponsoredProducts/sponsoredBrands） 4. 调用 /v2/sp/{campaignType}/report 创建报告任务 5. 轮询任务状态直到完成，下载 CSV 报表 6. 解析 CSV，提取每条广告的关键指标：Impression、Click、CTR、Spend、Sales、ACOS、CVR 7. 将解析后的数据上传到飞书多维表格（新建一个"广告周报"表格） 8. 同时将原始 CSV 作为飞书文件上传到飞书群 9. 返回"成功 / 失败"及摘要给 n8n
✅ 验收标准： n8n 触发后，飞书表格和群聊里出现上周的广告数据。

Dify 广告诊断 Agent

Dify
目标： 在 Dify 建一个广告分析 Agent，读取周报数据，输出优化建议。

Dify Agent 配置

创建新应用 → 选择"Agent"类型 → 命名"EMKE广告分析师"

知识库（Knowledge）：上传 EMKE 品牌广告基准指标文档
"EMKE 卫浴产品广告基准：LED浴室镜 ACOS 超过 35% 为异常（正常 15-25%）；暖气片 CVR 低于 2% 为异常（正常 3-5%）；淋浴房 CPC 超过 €1.5 需关注；马桶/陶瓷盆属于低频高客单价品类，CVR 偏低属正常。"

System Prompt：
"你是 EMKE 品牌的广告投放分析师。请分析输入的 Amazon 广告周报数据，输出以下内容：1) 本周 TOP 5 表现最好和最差的广告活动；2) 异常指标预警（ACOS > 35%、CVR < 2%、CPC 异常飙升）；3) 具体优化建议（格式： SKU XXX，关键词 YYY，当前 ACOS ZZ%，建议调整竞价至 XX 欧元，理由：…）；4) 需要暂停的无效关键词列表。输出语言为中文，建议可直接执行。"

工具（Tools）：不添加外部工具，纯分析型 Agent
✅ 验收标准： 输入一份真实周报 CSV 数据，Dify 输出包含具体竞价建议的分析报告。

人工确认 + 执行广告调整

n8n 飞书
目标： Dify 建议发到飞书 → 你审核 → 点击按钮确认 → n8n 自动调 API 修改广告。

n8n 工作流（两段）

第一段：生成建议
定时触发器（每周一早 8 点）→ HTTP Request 调用第 10 步的报表下载脚本 → 获取 CSV 数据 → HTTP Request 调用 Dify 广告诊断 Agent → 拿到分析结果 → 写入飞书"广告周报"表格的"最近广告建议"列 → 飞书机器人发消息通知你

第二段：执行调整
飞书多维表格触发器（"广告状态"列变为"已优化"）→ HTTP Request 调用 Trae 写的 Amazon Ads API 调价脚本 → 传入 Dify 建议里的具体参数（SKU、关键词ID、新竞价）→ 执行完成后更新飞书状态

你的日常操作（每周一 10 分钟）

  1. 打开飞书，查看"广告周报"表格里的 Dify 建议
  2. 高客单价产品（浴室柜、淋浴房）建议逐条看，低客单（水龙头配件）可批量通过
  3. 觉得某条建议合理 → 在飞书表格勾选"广告状态=已优化"
  4. n8n 自动触发执行脚本，去 Amazon 后台调价
✅ 验收标准： 完整链路跑通：周一自动出报告 → Dify 给建议 → 你审核 → 勾选后 Amazon 广告竞价自动修改。
